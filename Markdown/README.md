**Markdown 是一个 Web 上使用的文本到 HTML 的转换工具，可以通过简单、易读易写的文本格式生成结构化的 HTML 文档。目前 github、Stackoverflow 等网站均支持这种格式.**

### [官方文档](http://www.markdown.cn/#overview)

> Markdown 支持两种标题的语法，类 Setext 和类 atx 形式。

接下来简单描述一下语法：

### 1. 标题

> 类 Setext 形式是用底线的形式，利用 = （最高阶标题）和 - （第二阶标题） 这种形式被大多数的编辑器取消掉，默认会使用类 Atx 形式替换,不建议使用;
> 类 atx 的形式的标题使用 `# title` 来标记，`#` 与内容保留一个空格，否则不生效;当然可以选择性的闭合;
> 多一个`#`代表降一级（即字号小一级），总共有 6 级；如：

```
# 这是一级标题
## 这是二级标题、
...
###### 这是六级标题
```

效果如下：

- # 这是一级标题

* ## 这是二级标题、

- ### 这是三级标题

* #### 这是四级标题

- ##### 这是五级标题

* ###### 这是六级标题

---

### 2. 字体

- 斜体

  > 斜体的内容需要使用一个下划线 `_` 或者一个星号 `*` 进行包裹(部分编辑器会强制转换 `_` 包裹);
  > 标签与内容之间不能有空格;如果内容中含有相关标签内容需要使用 `\` 进行转义与编程语言相似;
  > 当页面加载时会被转成 `html` 中 `<em>`标签;

  例如： `_字体斜体_`

  效果： _字体斜体_

* 加粗

  > 加粗的内容需要使用两个星号 `**` 或者两个下划线 `__` 进行包裹(部分编辑器会强制转换 `**` 包裹)
  > 标签与内容之间不能有空格;如果内容中含有相关标签内容需要使用 `\` 进行转义与编程语言相似;
  > 当页面加载时会被转成 `html` 中 `<strong>`标签

  例如： `**字体加粗**`

  效果： **字体加粗**

- 斜体加粗

  > 斜体加粗的内容需要使用两个星号和一个下划线 `**_` 进行对称包裹;
  > 标签与内容之间不能有空格;如果内容中含有相关标签内容需要使用 `\` 进行转义与编程语言相似;

  例如： `**_斜体 \*\_\* 加粗_**`

  效果：**_斜体 \*\_\* 加粗_**

* 删除线

  > 增加删除线的内容需要使用两对波浪号 `~~` 进行包裹
  > 标签与内容之间不能有空格;如果内容中含有相关标签内容需要使用 `\` 进行转义与编程语言相似;

  例如：`~~字体删除线~~`

  效果：~~字体删除线~~

---

### 3. 区块引用

> 引用，就是描述内容标记为一个模块;Markdown 标记区块引用是使用类似 email 中用 > 的引用方式
> 区块里面可以嵌套使用其他的样式

例如：

```
> 1. 这是一段详细的描述...
> 2. 这是描述的内部的引用...
> > > > > > > 这是描述的内部的引用...
```

效果如下：

> 1.  这是一段详细的描述...
> 2.  这是描述的内部的引用...
>
> > > > > > > 这是描述的内部的引用...

---

### 4. 分割线

> 一条浅灰色的实线，使用三个及以上数量的星号`***`或者三个及以上数量减号`---`或者三个及以上数量下划线`___`进行标识,部分编辑器强制变更为减号，依据当前位置上下文环境；注意该行内不能有其他的元素

例如：

```
    ***
    ---
    ___
```

效果如下：

---

### 5. 图片

> 图片加载至支持使用网络图片,使用`![desc](image-url)`,和超链接有些相似;

例如：

```
![网络图片](http://inews.gtimg.com/newsapp_match/0/359390627/0)
```

效果如下：

![网络图片](http://inews.gtimg.com/newsapp_match/0/359390627/0)

---

### 6. 超链接

> 使用文字描述，支持点击进行跳转，使用 `[desc](url,"title")`,和图片有些相似;
> 当页面加载时会被转成 `html` 中 `<a>` 标签

例如：

```
[点击跳转](http://inews.gtimg.com/newsapp_match/0/359390627/0)
```

效果如下：

[点击跳转](https://typora.io/,"title")

---

### 7. 列表

- ##### 有序列表

  > 有序列表：使用 `1.`,`2.`,`3.`...排下去,注意序号和内容之间需要空格分隔，例如：

  ```
  1. title
  2. font
  3. image
  4. list
  ```

  效果：

  1.  title
  2.  font
  3.  image
  4.  list

- ##### 无序列表

  > 无序列表: `+`,`-`,`*`...排下去，不需要排序，部分编辑器会自动强制，注意序号和内容之间需要空格分隔，例如：

  ```
  - 超链接
  * 代码
  - 逻辑条件
  * 引用
  ```

  效果：

  - 超链接

  * 代码

  - 逻辑条件

  * 引用

---

### 8. 代码

> 代码有两种：单行代码和多行代码(两者的区别就是多行样式中区分间距和换行，保留代码块的样式，而单行则忽略);
> 单行代码：使用一对反向单引号 `` `包裹起来; 当页面加载时会被转成`html`中`p` 标签;

```
`This is a example`
```

> 效果如下：

`This is a example`

> 多行代码：使用三对反向单引号包裹起来;
> 保留样式的```效果如下(这种模式下支持多语言：c，java，php，js，python...几乎所有开发语言)：

````
```java // 这里指定代码的语言
  public static void main(String[] args){
      System.out.print("Hello world !")
  }
````

> 效果如下：

```java
  public static void main(String[] args){
     System.out.print("Hello world !")
  }
```

### 9. 表格

> 表格使用 `| tab | tab | tab |...|`声明表头字段
> 表格使用 `| :-- | :-: | --: |...|`申明表单中对应列中的内容对其方式,冒号在左侧即左对齐，在右侧即右对齐，在两侧即居中；
> 使用 `|value|value|value|...|`插入数据，例如：

```
| name | age | gender | user_info     |
| :--- | --: | :----: | :------------ |
| w    |  18 |   男   | 高 de 一批... |
| x    |  10 |   女   | 富 de 一批... |
| l    |  28 |   男   | 帅 de 一批... |
```

效果如下：

| name | age | gender | user_info     |
| :--- | --: | :----: | :------------ |
| w    |  18 |   男   | 高 de 一批... |
| x    |  10 |   女   | 富 de 一批... |
| l    |  28 |   男   | 帅 de 一批... |
